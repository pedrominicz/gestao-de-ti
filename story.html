User Stories

User Story 1 — Autenticação com JSON web token

Eu como líder do time de front-end gostaria de conseguir autenticar na API GraphQL para que o time de front-end possa acessar a API de forma segura.

Critérios de Aceitação
  • A API só deve ser acessível para usuários autenticados
  • A API não deve ter nenhum conceito de "conexão" que necessite que o front-end precise administrar
  • A autenticação deve acontecer por requisição
  • A autenticação deve ser feita por JSON web token
Definition of Done

O sistema de autenticação da API GraphQL foi testado por um member do time de front-end.

Estimativa

Esta atividade deve durar uma semana

User Story 2 — Performance da API

Eu como líder do time de front-end gostaria de a API GraphQL tivesse alta performance para que o time de front-end possa criar dashboards responsivos.

Critérios de Aceitação
  • A nova API GraphQL deve ser tão rápida quanto a antiga API REST
  • Consultas simples (uma tabela, menos de mil linhas) API GraphQL devem levar menos de 100 milissegundos
  • Consultas complexas (diversas tabelas) devem usar batching para otimizar consultas ao banco de dados e diminuir a latência
Definition of Done

Foi testado que a nova API GraphQL tem performance igual ou melhor que a antiga API REST nos principais dashboards.

Estimativa

Esta atividade deve durar duas semanas

User Story 3 — Acessar tabelas de controle

Eu como web developer gostaria de ter acesso às tabelas de unidades de controle (UC) e ordens de serviço (OS) do banco de dados através de uma API GraphQL para que consiga desenvolver novos recursos para o controle de operações sem depender do time de back-end.

Critérios de Aceitação

Conseguir acessar as seguintes tabelas:

  • OSEntrada
  • OSExecucao
  • OSSaida
  • UC
  • UCStatus
Definition of Done

Foi testado o acesso e modificação de dados das tabelas listadas através da API.

Estimativa

Esta atividade deve durar três semanas pela complexidade do modelo de dados nas tabelas envolvidas.

User Story 9 — Garantir o funcionamento da API GraphQL

Eu como líder do time de front-end gostaria de ter certeza que a nova API GraphQL é estável para que o time de front end possa usá-la sem risco de instabilidade.

Critérios de Aceitação
  • Todas as queries já usadas no front end devem ser testadas
  • Todas as queries com índices devem ser testadas com valores inválidos
  • Todas as queries com parâmetros devem ser testadas com valores inválidos
Definition of Done

Foi testado o acesso e modificação de dados de todas às tabelas, inclusive quando o servidor está sob alto estresse.

Estimativa

Esta atividade deve durar uma semana
